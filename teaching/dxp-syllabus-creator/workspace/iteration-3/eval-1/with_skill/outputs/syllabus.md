## 一、课程的性质、任务和目标

### 1. 课程性质

《数据结构与算法》是智能科学与技术专业的一门学科基础必修课。课程系统讲授常用数据结构的组织原理、存储表示及其算法设计与分析的基本方法，是连接程序设计基础与专业核心课程的桥梁。本课程以讲授为主，辅以上机实践训练，属于理论课程。

### 2. 课程任务

本课程的任务是使学生掌握线性表、栈与队列、串、数组与广义表、树与二叉树、图等基本数据结构的逻辑特征、存储实现及基本操作，理解查找和内部排序的经典算法及其性能分析方法，具备针对实际问题选择合适数据结构和设计高效算法并编程实现的能力。通过系统的上机实践训练，培养学生良好的程序设计习惯、算法调试能力与计算思维素养，为后续专业课程的学习以及从事智能系统开发工作奠定坚实的基础。

### 3. 课程目标

**课程目标1（知识目标）：** 系统掌握各类常用数据结构（线性表、栈、队列、串、树、图等）的逻辑定义、存储结构和基本操作的实现方法，理解查找和排序的经典算法原理及其时间复杂度分析方法。

**课程目标2（能力目标）：** 能够针对具体的应用问题，分析数据的组织方式和操作需求，选择合适的数据结构并设计高效的算法，能够用C语言编程实现并进行测试验证。

**课程目标3（素养目标）：** 培养抽象思维能力、算法效率优化意识和计算思维习惯，养成严谨规范的程序设计风格，具有良好的工程伦理意识和团队协作精神，能够客观评价算法性能并持续改进。

## 二、课程教学内容的基本要求、重点和难点及学时分配

### 第1章 绪论（4学时，讲课4学时）

**基本要求：** 理解数据、数据元素、数据项、数据结构、数据类型、抽象数据类型等基本概念；掌握算法及其时间复杂度、空间复杂度的分析方法；了解数据结构课程在智能科学专业课程体系中的地位。

**重点和难点：**
- 重点：数据结构的基本概念和术语，算法时间复杂度的渐近分析方法
- 难点：利用大O记号对算法进行时间复杂度分析

**教学方法：** 课堂讲授为主，结合实例演示复杂度分析过程。

### 第2章 线性表（8学时，讲课6学时 + 上机2学时）

**基本要求：** 理解线性表的逻辑结构特征和抽象数据类型定义；掌握线性表的顺序存储结构（顺序表）和链式存储结构（单链表、循环链表、双向链表）的实现方法及基本操作；能够比较两种存储结构的优缺点和适用场景；掌握线性表基本操作（插入、删除、查找）在两种存储结构上的算法实现与复杂度分析。

**重点和难点：**
- 重点：顺序表与单链表的存储表示及基本操作实现，头指针和头结点的区别
- 难点：单链表中插入和删除操作中指针的正确修改，双向链表的操作

**上机内容：** 实现单链表的基本操作（创建、插入、删除、遍历），并用测试数据验证。

**教学方法：** 课堂讲授+代码演示+上机编程实践。

### 第3章 栈与队列（8学时，讲课6学时 + 上机2学时）

**基本要求：** 掌握栈和队列的逻辑结构特点、各自的抽象数据类型定义；掌握顺序栈、链栈、循环队列、链队列的存储表示和基本操作实现；理解栈在递归算法、表达式求值中的应用，理解队列在广度优先搜索中的应用。

**重点和难点：**
- 重点：栈和队列的逻辑特征（LIFO/FIFO）、存储实现及基本操作
- 难点：循环队列中队空和队满的判别条件，栈在递归消除中的作用机理

**上机内容：** 实现顺序栈和循环队列的基本操作，利用栈解决表达式求值问题。

**教学方法：** 课堂讲授+动画演示+上机编程实践。

### 第4章 串、数组与广义表（6学时，讲课4学时 + 上机2学时）

**基本要求：** 理解串的存储结构和基本操作，掌握KMP模式匹配算法；掌握数组的存储表示和特殊矩阵的压缩存储方法；理解广义表的逻辑结构和存储表示。

**重点和难点：**
- 重点：KMP模式匹配算法中next数组的计算，特殊矩阵的压缩存储
- 难点：KMP算法的理解与实现，广义表的递归处理

**上机内容：** 实现KMP字符串匹配算法，实现稀疏矩阵的转置操作。

**教学方法：** 课堂讲授+算法动态可视化演示+上机编程实践。

### 第5章 树与二叉树（12学时，讲课8学时 + 上机4学时）

**基本要求：** 理解树的定义和基本术语；掌握二叉树的性质、存储结构和遍历算法（前序、中序、后序、层次）；掌握哈夫曼树与哈夫曼编码的构造方法及应用；理解树、森林与二叉树之间的转换；了解平衡二叉树、B树等高级树结构的基本概念及应用场景。

**重点和难点：**
- 重点：二叉树的遍历算法（递归与非递归实现），哈夫曼树的构造与编码
- 难点：二叉树的非递归遍历实现，哈夫曼编码的生成过程

**上机内容：** 实现二叉树的建立与三种遍历算法；实现哈夫曼编码的全过程（建树、编码、解码）。

**教学方法：** 课堂讲授+算法可视化演示+分组讨论+上机编程实践。

### 第6章 图（10学时，讲课8学时 + 上机2学时）

**基本要求：** 理解图的基本概念和术语；掌握图的邻接矩阵和邻接表两种存储结构；掌握图的深度优先搜索和广度优先搜索算法；掌握最小生成树（Prim算法、Kruskal算法）、最短路径（Dijkstra算法、Floyd算法）、拓扑排序和关键路径等经典图算法；了解图算法在智能系统（如知识图谱、路径规划）中的应用。

**重点和难点：**
- 重点：图的遍历算法和最短路径算法（Dijkstra算法）
- 难点：最短路径算法中贪心策略的理解，关键路径中最早/最晚开始时间的计算

**上机内容：** 实现图的邻接矩阵/邻接表存储，实现深度优先和广度优先搜索遍历。

**教学方法：** 课堂讲授+算法动画演示+应用案例分析+上机编程实践。

### 第7章 查找（8学时，讲课6学时 + 上机2学时）

**基本要求：** 理解查找的基本概念和评价指标（查找长度）；掌握顺序查找、折半查找、分块查找的原理与性能分析；掌握二叉排序树的构造和查找；理解平衡二叉树（AVL）的平衡调整原理；掌握哈希表的构造方法、冲突处理策略和查找性能分析。

**重点和难点：**
- 重点：折半查找算法、二叉排序树的构造与查找、哈希表的建立与查找
- 难点：哈希函数的设计与冲突处理策略选择，平衡二叉树的调整

**上机内容：** 实现二叉排序树的建立和查找；实现哈希表的建立（链地址法和开放定址法）及查找。

**教学方法：** 课堂讲授+分步演示+对比分析+上机编程实践。

### 第8章 内部排序（8学时，讲课6学时 + 上机2学时）

**基本要求：** 掌握插入排序（直接插入排序、希尔排序）、交换排序（冒泡排序、快速排序）、选择排序（简单选择排序、堆排序）、归并排序和基数排序的基本思想和算法实现；能够对各种排序算法进行时间复杂度与空间复杂度分析；理解不同排序算法的稳定性、适用场景及算法选择的依据。

**重点和难点：**
- 重点：快速排序、堆排序和归并排序的算法实现与性能分析
- 难点：堆排序中建堆和调整堆的过程，快速排序的递归实现与性能分析

**上机内容：** 实现快速排序和堆排序，对比不同排序算法在不同数据规模下的性能表现。

**教学方法：** 课堂讲授+动画对比演示+性能实验+上机编程实践。

### 2. 过程考核

**（1）平时作业（权重15%）**

每章布置2-4道课后习题或算法设计题，要求学生独立完成并按时提交。作业内容涵盖基础概念辨析、数据结构操作的手工模拟和简单算法设计，重点检验学生对课程目标1（知识目标）的掌握程度。

评分标准：
- 优（90-100）：作业按时提交，解答完整正确，算法设计思路清晰，代码规范
- 良（80-90）：作业按时提交，解答基本正确，算法设计合理，有少量细节问题
- 中（70-80）：作业基本按时提交，解答存在少量错误，算法思路基本正确
- 及格（60-70）：作业偶有迟交，解答存在较多错误，算法实现不完整
- 不及格（0-60）：作业严重缺交或大量错误，算法思路混乱

**（2）课堂测验（权重10%）**

在学期中和学期末各安排一次课堂小测验，每次约20分钟，题型为选择题和简答题，主要考查学生对核心概念、经典算法原理的理解（对应课程目标1）。

**（3）上机实验（权重25%）**

共安排8次上机实验（每次2学时），对应第2-8章的教学内容。要求学生独立完成实验任务，提交源代码和实验报告。实验评价包括功能正确性、代码规范性、算法效率分析和实验报告质量四个方面（对应课程目标2和课程目标3）。

评分标准：
- 优（90-100）：程序功能完整正确，代码结构清晰规范，报告分析深入，有创造性拓展
- 良（80-90）：程序功能正确，代码较规范，报告分析合理
- 中（70-80）：程序基本功能正确，代码可读性一般，报告分析基本完整
- 及格（60-70）：程序存在少量功能缺陷，代码不够规范，报告分析简略
- 不及格（0-60）：程序无法运行或严重偏离实验要求，报告缺失或质量极差

### 3. 期末考试

期末考试采用闭卷笔试形式，考试时间120分钟，卷面满分100分。试卷结构：

| 题型 | 分值 | 考核要点 |
| :--- | :---: | :--- |
| 选择题 | 20分 | 基本概念、数据结构性质 |
| 填空题 | 10分 | 核心术语、算法步骤 |
| 简答题 | 20分 | 数据结构操作过程、算法原理 |
| 综合应用题 | 30分 | 算法设计、数据结构选择、综合问题分析 |
| 编程题 | 20分 | 数据结构的算法实现 |
| **合计** | **100分** | |

期末考试评分标准：
- 优（90-100）：数据结构概念理解透彻，算法设计创新高效，编程实现完整正确
- 良（80-90）：概念理解正确，算法设计合理，编程实现基本正确
- 中（70-80）：概念基本正确，算法设计有少量疏漏，编程实现存在部分缺陷
- 及格（60-70）：概念部分掌握，算法设计有较多不足，编程实现存在明显缺陷
- 不及格（0-60）：基本概念不清，算法设计混乱，编程无法正确表达

## 五、有关说明

1. **先修衔接：** 本课程以《高级语言程序设计（C语言）》中C语言基本语法、函数、指针、结构体等知识和《离散数学》中集合论、图论基础、数理逻辑为前置知识基础。学生须具备基本的C语言程序设计与调试能力，熟悉结构化程序设计思想，具备初步的数学抽象思维和逻辑推理能力。

2. **后续承接：** 本课程为后续《操作系统》《数据库原理》《算法设计与分析》《机器学习》《数据挖掘》《计算机视觉》《自然语言处理》等课程提供数据结构选择、算法设计实现与复杂性分析等核心能力支撑。特别是图论算法部分直接为《机器学习》中的图模型学习、最短路径算法等提供方法基础。

3. **开课学期建议：** 建议在第3学期（大二上学期）开设。前两个学期学生已修完《高级语言程序设计》和《离散数学》，具备了必要的编程与数学基础；同时在第3学期开设本课程，可为第4-6学期的专业核心和专业选修课的学习奠定算法基础。

4. **修读建议：** 本课程理论性与实践性并重，建议学生在课堂学习之外，每周投入不少于6小时的课外学习时间（其中编程实践不少于4小时）。鼓励利用LeetCode、PTA、VisuAlgo等平台进行课外算法训练和可视化学习。上机实验须独立完成，严禁抄袭，教师通过代码查重工具进行学术诚信检查。

5. **课程适用对象：** 本课程适用于智能科学与技术专业大二本科生自动修读。其他相关专业（计算机科学与技术、人工智能、数据科学与大数据技术等）学生若需修读本课程，须提交申请并经开课学院审批。

## 六、课程建设与改革摘要

### 1. 立德树人

本课程深入挖掘数据结构与算法知识体系中的思政育人元素，将价值引领融入专业知识教学全过程，实现为党育人、为国育才的根本目标。具体思政融入方案包括：（1）在绪论章节中，结合我国在操作系统（华为鸿蒙）、数据库（OceanBase、TiDB）和人工智能大模型（DeepSeek）等基础软件领域的自主创新成就，引导学生树立科技自立自强的家国情怀和使命担当，理解数据结构与算法作为基础软件基石的战略意义。（2）在查找与排序章节中，通过算法效率优化的演进历程，培养学生的精益求精的工匠精神和追求卓越的创新素养。（3）在图论章节中，结合最短路径算法在智慧交通、物流调度等公共服务领域的社会价值，增强学生以技术服务社会的责任感。（4）在上机实验环节中强调学术诚信和代码原创性，培育严谨求实的科学精神和职业道德。预期成效：学生能够在掌握专业知识的同时，形成正确的科技伦理观和创新价值观，增强以国家战略需求为导向的问题意识。

### 2. 教学思想

本课程坚持"以学生为中心、以产出为导向、以质量为核心"的OBE教育理念。课程目标依据智能科学与技术专业毕业要求中的"工程知识""设计/开发解决方案""使用现代工具"三个核心毕业要求指标点反向设计，确保每一个知识模块和能力训练环节都有明确的目标指向。在教学中落实因材施教原则，通过分层布置作业（基础题+提高题+挑战题）满足不同层次学生的学习需求，鼓励学有余力的学生参与算法竞赛和科研项目。面对AI时代对人才培养提出的新挑战，本课程在保持经典算法理论体系完整性的同时，引导学生利用AI编程辅助工具（如GitHub Copilot、CodeGeeX）提升开发效率，同时强调对AI生成代码的批判性审查和效率优化能力，实现"人机协作"环境下计算思维能力的升级，而非简单的代码生成依赖。将知识传授（数据结构原理）、能力培养（算法设计与编程实现）与价值引领（科技伦理与家国情怀）有机统一于教学全过程。

### 3. 教学方法

本课程构建"理论讲授+编程实践+创新探索"三维教学模式，综合运用以下四种核心教学方法：（1）**课堂理论讲授**（用于各章概念原理教学）：在绪论、线性表、树与二叉树等核心章节的理论讲解中，配合VisuAlgo算法可视化平台进行动态演示，帮助学生建立数据结构的直观认知。（2）**案例驱动教学**（用于图的遍历、最短路径等应用性内容）：选取智慧校园导航、社交网络分析等贴近学生生活的真实案例，引导学生从问题抽象、数据结构选择、算法设计到编程实现的完整流程中掌握计算方法论。（3）**项目驱动教学**（用于上机实验的综合训练环节）：设置开放性实验项目（如简易文件系统索引、校园地图最短路径查询、学生成绩管理系统等），3-4人为一组协作完成，培养学生的团队协作与系统开发能力，期末进行项目展示与答辩。（4）**翻转课堂+在线评测**（用于查找、排序等算法对比性内容）：课前发布预习任务和在线自测题，课中组织小组讨论和算法对比分析，课后在LeetCode/PTA平台完成在线算法习题，利用自动评测系统获得即时反馈。通过上述方法的组合运用，激发学生的学习主动性和创造力。

### 4. 教学手段

本课程以智慧树/雨课堂智慧教学平台为核心，构建"课前-课中-课后"全流程智能化教学生态。（1）**课前智能预习：** 通过平台推送预习课件和微视频，发布预习题检测学生预习效果，教师根据预习数据精准调整课堂教学重点。（2）**课中互动探究：** 利用雨课堂的实时答题、弹幕互动和随机点名功能增强课堂参与度，借助VisuAlgo（https://visualgo.net）和Data Structure Visualizations（https://www.cs.usfca.edu/~galles/visualization/）等算法可视化工具进行动态演示，将抽象的算法过程直观呈现。（3）**课后精准复盘：** 基于PTA程序设计实验辅助教学平台的自动评测功能，实现作业和实验的即时批改与反馈；利用平台的教学过程数据进行学习行为分析，识别学习困难学生并实施精准帮扶。（4）**AI辅助教学工具：** 引入GitHub Copilot辅助学生代码编写，使用CodeGeeX进行代码审查和优化建议，同时利用自建的算法题库AI组卷系统进行个性化考核。平台留存的学生学习全过程数据（课堂互动记录、作业成绩趋势、实验完成质量、在线评测轨迹等）经脱敏处理后用于教学诊断与持续改进，以数据驱动的方式提升教学质量和育人效能。
