## 1. 页眉表（Table 0）替换规则

模板的页眉表已预填了大量标签文字（如"课程编码""课程名称""学分"等）。替换时遵循以下通用原则：

### 标签单元格识别与保护

**什么是标签单元格**：单元格文本是字段名称（如"课程编码""课程名称""学分""总学时""讲课""实验""上机""实践""考核方式""执行学期""先修课程""后续课程""课程类别""课程负责人"等）。

**保护规则**：
- 标签单元格保持原样，不覆写、不清空
- 通过检查单元格文本是否匹配已知标签来判断是否为标签单元格
- 若为标签单元格则跳过，否则为值单元格，可以写入数据

### 值单元格填充策略

**通用方法**：遍历页眉表所有行和列，对每个单元格：
1. 检查文本是否为已知标签 → 跳过
2. 检查文本是否为空或仅含占位符 → 写入对应的课程数据
3. 检查文本是否为预填的默认值（如考核方式默认描述）→ 可整体替换

### "课程负责人"区域绝对保护

模板中有一组"课程负责人"标签及其值区。这两个区域均不得以任何方式写入或清空——标签一旦被覆写会破坏模板结构，值区应留给用户手工填写。

**实现方法**：在代码中识别包含"负责人"文本的单元格及其同行右侧的所有单元格，全部跳过。

### 学时值独立写入

总学时、讲课、实验、上机、实践五个学时值必须各自独立写入纯数字值。禁止合并写入（如"64（讲课48学时，实验16学时）"）。每个值单元格只接受一个数字。

**原因**：培养方案中这五项是独立字段，分开显示便于教务系统录入和审核比对。

## 2. 评分标准表替换规则

评分标准表（平时作业/课堂测验/实验/期末考试的评分标准）是模板的核心表格。

### 表结构

每表结构为 3 列：课程目标 | 评价等级 | 评分标准。行数取决于课程目标数 × 5 个评分等级 + 1 行表头。

### 数据构造规则

构造数据数组时，将表头 `['课程目标', '评价等级', '评分标准']` 作为数组的第 0 个元素（索引 0），然后从索引 1 开始填充评分数据。这样在循环写入时不会覆盖模板已有的表头。

### 评价等级列（第 2 列）

只允许包含以下 5 种固定文本：

```
优 90-100
良 80-90
中 70-80
及格 60-70
不及格 0-60
```

- 禁止在此列中出现任何评分标准描述文字
- 禁止在此列中出现课程目标编号、章节名称等其他信息
- 各行的等级单元格独立设置

### 评分标准列（第 3 列）

- 内容必须为针对当前课程的原创描述，使用课程专有术语
- 禁止直接复制模板中的原课程评分描述文字
- 每个等级对应一个评分标准描述，各等级之间体现递进差异

### 课程目标列（第 1 列）

- 目标编号必须使用"课程目标N"格式（含"课程"前缀）
- 与 Markdown 大纲中定义的课程目标编号完全一致
- 若课程目标数少于模板行数，多余行清空

## 3. 正文段落替换策略

### 核心原则

模板中的占位段落（如"……"或括号标注文字）必须用已生成的 Markdown 大纲中的课程专属内容完全替换。仅靠全局文本替换（如将原课程名替换为新课程名）无法将模板占位内容变为当前课程的实质性描述。

### 文本特征匹配

不依赖段落精确索引号（不同课程内容长度不同，索引会变化）。改用文本特征匹配：
1. 遍历 `doc.paragraphs`，检查每个段落的 `text.strip()`
2. 匹配特征文本（如"课程性质""课程任务""课程目标""基本要求""重点和难点""教学方法""建议教材"等）
3. 找到后，替换其后的第一个非空段落的文本

### 段落替换方法

```python
p.clear()        # 清除所有 runs
add_run_songti(p, new_text)  # 写入新文本，同时设置宋体字体
```

**关键**：`p.clear()` 后新创建的 run 不会继承模板的字体设置。必须使用显式设置宋体字体的辅助函数（如 `add_run_songti()`）。

### Footer 保护

模板末尾有"编写人/审核人/批准人/编写日期"四行 footer。做任何段落替换时，如果段落文本以这些关键词开头，则跳过所有替换操作。这些内容保持模板原样，由用户手工填写。

## 4. 多章节段落插入方法

### 问题

模板中第二节（教学内容）只有一组章节占位段落（1 个章节标题 + 基本要求 + 内容 + 重点和难点 + 内容 + 教学方法 + 内容）。当课程有多个章节时，需要为第 2 章及之后的章节插入新的段落。

### 解决方案

使用 `insert_paragraph_before(ref_p, text)` 辅助函数，在"……"占位段落之前插入新章节的段落：

```python
# 找到"……"占位段落
for i, p in enumerate(doc.paragraphs):
    if p.text.strip() == '……':
        # 在此之前插入所有额外章节
        for chapter in chapters[1:]:
            insert_paragraph_before(p, chapter['heading'], bold=True)
            insert_paragraph_before(p, "基本要求")
            insert_paragraph_before(p, chapter['req'])
            insert_paragraph_before(p, "重点和难点")
            insert_paragraph_before(p, chapter['key'])
            insert_paragraph_before(p, "教学方法")
            insert_paragraph_before(p, chapter['met'])
            insert_paragraph_before(p, "")  # 空行分隔
        # 清除"……"占位
        p.clear()
        break
```

### 实现原理

`insert_paragraph_before` 通过深拷贝参考段落的 XML 元素并清除其内容，然后用 `addprevious` 插入到目标位置之前。新段落的文本通过 `add_run_songti()` 写入，确保字体一致。

## 5. 表格行数匹配

模板中各表格的行数是固定的。填充数据时确保数据行数不超过表格可用行数：

| 表 | 总行数 | 表头行 | 可用数据行 | 常见内容 |
|:---|:---:|:---:|:---:|:---|
| 课程目标矩阵 | 5 | 1 | 4 | 4个课程目标 |
| 教学内容对应 | 6 | 1 | 5 | 5个教学模块 |
| 平时作业 | 6 | 1 | 5 | 5次作业 |
| 平时作业评分 | 16 | 1 | 15 | 3目标×5等级 |
| 课堂测验 | 4 | 1 | 3 | 3次测验 |
| 测验评分标准 | 16 | 1 | 15 | 3目标×5等级 |
| 实验对应 | 4 | 1 | 3 | 3次实验 |
| 实验评分标准 | 6 | 1 | 5 | 1目标×5等级 |
| 期末考试对应 | 5 | 1 | 3或4 | 3-4个内容模块 |
| 期末评分标准 | 16 | 1 | 15 | 3目标×5等级 |
| 达成度评价 | 7 | 2 | 4 | 3目标+合计行 |

- 若教学内容超过可用行数，合并章节到同一行
- 若课程目标数少于模板行数，多余数据行必须显式清空（`cell.text = ''`）
- 实验次数多于 3 次时，合并为 3 组写入

### Table 2 学时分配规则

Table 2 中"学时"列的值之和必须等于总学时。**关键规则**：讲课学时和实验学时分离到不同行，不要在同一行中混合计入。

推荐做法：
- 前 N 行：各教学模块的**讲课学时**（纯数字）
- 最后 1 行：**实验学时合计**（纯数字）
- 所有行学时之和 = 总学时

避免的做法：将实验学时分散到各章节行中，同时又有一个实验合计行——这会导致学时重复计算。

## 6. 列宽调整

所有表格在内容替换完成后，根据内容长度调整列宽：

- **评分标准表**：评分标准列 ≥ 8cm，评价等级列 2-3cm，课程目标列 2.5-3.5cm
- **教学内容表**：课程教学内容列 ≥ 9cm（长文本需要足够宽度）
- **数据对应表**（作业/测验/实验/期末）：内容列 ≥ 7cm
- **页眉表**：有合并单元格，列宽基本固定，不做大幅调整

使用 `cell.width = Cm(xx)` 对每列的首行单元格设置宽度。

## 7. 常见陷阱与解决方案

### 陷阱 1：正文段落替换覆盖了错误的段落

**现象**：替换"课程性质"正文时，跳过了空段落，覆盖了"课程任务"或"课程目标"子标题。

**原因**：模板中子标题后的正文段落可能是空的。如果搜索"下一个非空段落"来替换，会跳过空段落并找到错误的段落。

**解决方案**：直接替换子标题后的第一个段落（索引 i+1），无论是否为空。不要搜索"下一个非空段落"。

### 陷阱 2：全局文本替换破坏出版社名称

**现象**：将"机械"替换为"计算机"后，"机械工业出版社"变成了"计算机工业出版社"。

**原因**：全局替换范围过大，替换单字而非完整课程名称。

**解决方案**：全局替换只替换**课程全名称**（如"机械原理"→"新课程名"），绝不替换单字或部分名称。

### 陷阱 3：中文字体不一致

**现象**：生成的 docx 中，某些段落字体不是宋体，与其他正文不一致。

**原因**：`p.clear()` 清除了所有 runs 及其格式，新 `add_run()` 创建的 run 没有字体设置。

**解决方案**：所有 `add_run()` 调用后，使用辅助函数显式设置 `font.name='宋体'` 和 `eastAsia='宋体'`。

### 陷阱 4：Python 字符串中的中文引号

**现象**：Python 脚本报 SyntaxError，指向包含中文引号的行。

**原因**：中文引号""（U+201C/U+201D）在某些编码下可能与 ASCII 双引号混淆。

**解决方案**：
- 使用 `\u201c` 和 `\u201d` 转义序列代替直接输入中文引号
- 或使用三引号字符串（`"""..."""`）
- 或在文件头部声明 `# -*- coding: utf-8 -*-`

### 陷阱 5：PowerShell 编码问题

**现象**：在 PowerShell 中运行 Python 脚本时，中文输出乱码。

**原因**：PowerShell 默认使用 GBK 编码，而 Python 脚本输出 UTF-8。

**解决方案**：
- 在脚本头部添加 `import sys, io; sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')`
- 或使用 `chcp 65001` 切换 PowerShell 到 UTF-8
- 避免在 PowerShell 中内联 Python 代码（f-string 中的引号会与 PowerShell 冲突），改为写入 .py 文件后执行

### 陷阱 6：评分标准表表头被覆盖

**现象**：评分标准表的表头行变成了数据内容。

**原因**：循环写入数据时从 row 0 开始，覆盖了表头。

**解决方案**：构造数据数组时将表头作为第 0 个元素，循环从 row 0 开始写入，表头会被相同文本覆盖（无影响），数据从 row 1 开始正确填充。
