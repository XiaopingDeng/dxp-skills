# 培养方案解析指南

从 2026 版培养方案 `.docx` 中提取课程教学大纲所需元数据。

## 提取流程

### 1. 用 python-docx 打开文件

```python
import docx
doc = docx.Document("培养方案文件路径.docx")
```

### 2. 从段落中读取专业信息

读取所有段落，寻找：
- **专业名称**：第一段通常是 "XXX专业人才培养方案"
- **专业代码**：如 "080907T"
- **主干学科/相关学科**
- **主要课程列表**（看是否包含目标课程名称）

### 3. 从表格中读取课程元数据

课程计划表（通常是最大的表格，列数 15-20）包含每门课的详细信息。

对于目标课程，提取以下列数据：

| 字段 | 说明 |
| :--- | :--- |
| 课程编码 | 如 XD26040008 |
| 课程名称 | 如 计算机网络 |
| 学分 | 如 3 |
| 考核方式 | 考试 或 考查 |
| 总学时 | 如 52 |
| 讲课学时 | 如 44 |
| 实验学时 | 如 8（可能列名为"实验"或"实践"） |
| 上机学时 | 如 0 |
| 建议开课学期 | 看周学时所在的学期列位置 |

**判断课程性质和类别**：看表格中的"性质"和"类别"列：
- 公共必修 / 学科基础必修 / 专业基础必修 / 专业必修
- 专业限选 / 专业任选

### 4. 从毕业要求矩阵表读取支撑关系

毕业要求与课程关系矩阵表通常标注每门课程对各毕业要求的支撑强度（H/M/L）。

找到目标课程所在行，读取各列的值：
- **H**（高支撑）、**M**（中支撑）、**L**（弱支撑）
- 空的表示无直接支撑关系

将这些值映射到对应的毕业要求编号和描述。

### 5. 提取先修课程

从课程计划表中，找出排在该课程之前的同专业核心课程作为先修课程。

## 输出示例

```python
course_info = {
    "code": "XD26040008",
    "name": "计算机网络",
    "category": "专业必修课",
    "credits": 3,
    "assessment": "考试",
    "total_hours": 52,
    "lecture_hours": 44,
    "lab_hours": 8,
    "semester": 5,
    "prerequisites": ["C语言程序设计基础", "数据结构与算法", "计算机组成原理"],
    "graduation_requirements": {
        "要求2": "H",   # 问题分析
        "要求3": "M",   # 设计/开发解决方案
        "要求5": "H",   # 使用现代工具
    }
}
```
