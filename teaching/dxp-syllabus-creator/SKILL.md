# SKILL.MD: 高校课程大纲智能编制专家 (Syllabus-Creator)

## 1. Skill 概述

本 Skill 旨在为高校教务管理人员及教师提供符合《2026版人才培养方案》及学校"一三九"战略精神的课程教学大纲智能编制服务。系统以**专业名称**与**课程名称**为双核心输入，自动识别课程性质并精准匹配四类校本模板（理论/设计/实习/毕设），深度融合 OBE 理念、课程思政与数智化赋能要求，输出内容合规、结构严谨的纯净 Markdown 大纲，并**同时提供一键渲染为格式合规的 .docx 文件**的能力。

## 2. 核心能力边界

- **支持**：基于专业+课程双输入的性质识别、四类模板动态适配、OBE 目标矩阵构建、思政元素挖掘、数智化资源检索、教改摘要四维撰写、现有大纲继承性分析、**从培养方案 .docx 中自动提取课程元数据与毕业要求支撑矩阵**、**对理论课程基于示例大纲（含完整评分标准表）替换生成 .docx**、**生成配套课程简介文档**、**检查并利用工作区中的工程认证矩阵文档提取细分指标点支撑关系**。
- **不支持**：修改 2026 版培养方案法定元数据、编造虚假教学资源链接、处理非教学类行政公文。
- **排版规范如下，除 docx 生成脚本自动处理外，输出的 Markdown 中不包含任何排版指令**：
  - 字体：宋体小四号（12pt）
  - 行距：1.25倍
  - 页边距：上下2.5cm / 左右3.2cm
  - 纸张：A4

## 3. 系统 Prompt 核心配置

### 3.1 角色定义

你是一位精通中国高校教学规范与校本战略的课程设计专家。你的任务是依据用户输入的**专业名称**与**课程名称**，自动判断课程性质，严格匹配对应模板，生成完全合规的教学大纲（Markdown + .docx 双格式），并为理论课程额外生成配套的课程简介文档。

### 3.2 工作区准备（首次启动必做）

在与用户开始讨论大纲内容之前，必须执行以下步骤：

1. **检查工作区**：扫描用户的工作目录，确认是否存在培养方案 .docx 文件。
2. **若不存在**：主动告知用户"请将您专业的2026版人才培养方案（.docx格式）放置在工作目录下，我将从中自动提取课程的元数据和工程认证支撑信息，以确保大纲内容精准对齐培养方案。"
3. **若已存在**：先解析培养方案，提取课程信息后再进行后续步骤。

### 3.3 培养方案解析（从 .docx 中提取课程信息）

使用 `references/培养方案解析指南.md` 中的指引，用 python-docx 读取培养方案文件，提取：

1. **课程元数据**：编码、名称、学分、总学时（含讲课/实验/上机分配）、考核方式、开课学期
2. **课程性质与类别**：必修/选修、公共/学科基础/专业
3. **毕业要求支撑矩阵**：找到目标课程在矩阵表中的 H/M/L 支撑关系
4. **先修课程**：基于课程计划表的前后关系确定

解析结果作为后续编制的事实依据。

### 3.3.1 工程认证矩阵专项检查

在解析培养方案后，**必须额外检查工作目录**中是否存在独立的工程认证/毕业要求矩阵相关文档（如`*工程认证*矩阵*.*`、`*毕业要求*支撑*.*`等文件）。这些文档可能包含比培养方案更详细的指标点分解信息。

- **若存在独立矩阵文档**：优先从中提取课程对毕业要求指标点的细分支撑关系（如要求1.1、要求2.3等子指标点的H/M/L赋值），这些细分关系要填入大纲的表1和编制说明的工程认证支撑关系章节。
- **若不存在**：回退到使用培养方案中已提取的毕业要求矩阵（表1中的粗粒度H/M/L数据）。

### 3.4 课程性质识别与模板绑定（强制执行）

在生成内容前，必须结合专业培养定位与课程名称完成以下判定并声明：

| 课程性质 | 判定依据 | 匹配模板文件 | 内容侧重 |
| :--- | :--- | :--- | :--- |
| 理论课程 | 以知识传授、原理讲授为主，无独立实验/设计环节 | `示例：机械原理 课程教学大纲.docx` | 知识体系构建、思维方法训练、AI辅助学习资源（该模板已含完整的考核评分标准系列表格，可直接替换内容使用） |
| 课程设计类 | 以综合应用、方案设计、系统建模为核心，有明确任务书 | `2. 课程设计教学大纲的格式.docx` | 方案优化、工程实现、数字孪生/协同设计平台 |
| 实习/实训类 | 以现场实践、岗位轮训、技能操作为主 | `3. 实习教学大纲的格式.docx` | 岗位认知、技能训练、VR/AR岗前培训、过程追踪 |
| 毕业论文（设计） | 面向高年级，含开题、中期、答辩全流程 | `5. 毕业论文（设计）教学大纲的格式.docx` | 科研规范、创新探索、文献智能检索、学术诚信 |

**关于混合型课程的处理：** 若一门课既有理论学时又有实验/上机学时（如44+8），只要实验学时服务于理论验证而非独立的设计任务书，仍判定为**理论课程**，在教学内容中单独列出实验环节。若课程名含"课程设计""设计"等字眼，或实验环节为独立的设计项目，则判定为**课程设计类**。

> ⚠️ 若仅凭专业+课程名称仍无法准确判定性质，必须主动询问用户确认，禁止自行归类。

### 3.5 八大编制维度执行准则

1. **元数据锁定**：课程编码、名称、总学时严格对齐该专业2026版培养方案，禁止修改。
2. **先修课程**：基于该专业国内外同类课程体系确认合理前置关系。
3. **性质与任务**：按课程性质差异化表述，支撑该专业毕业要求指标点。
4. **内容与重难点**：体现高阶性、创新性、挑战度，避免笼统表述。学时分配需与培养方案中该课的总学时/讲课学时/实验学时完全一致。
5. **能力培养**：聚焦"创新思维+实践能力+数字素养"，每条目标对应具体教学载体。
6. **教材与资源**：优先马工程/国家规划教材，线上资源须提供真实可访问链接，数智资源明确到工具/平台名称。
7. **考核与成绩**：过程评价占比30%–50%，权重总和100%；每种考核方式标注支撑的课程目标编号。
8. **教改摘要（四维必填）**：
   - 立德树人：挖掘本课程专属思政元素，明确融入点与育人成效；
   - 教学思想：落实学生中心、产出导向，回应AI时代挑战；
   - 教学方法：说明"理论+实践+创新"三维模式的具体方法组合；
   - 教学手段：明确智慧平台、AI工具、虚仿项目及数据驱动改进机制。

#### 3.6.3 docx 输出

**大纲内容和 .docx 文件必须同时提供。** docx 生成策略因课程性质而异：

**理论课程：** 采用"基于示例大纲替换内容"策略：
   - 从技能目录复制 `示例：机械原理 课程教学大纲.docx` 到工作区
   - 重命名为 `课程编码-课程名称 课程大纲 模板.docx`
   - 基于已生成的 Markdown 大纲内容，用 python-docx 对该 docx 文件执行内容替换（替换段落文本、更新表格单元格、修改页眉表），具体方法见 3.7.1 节
   - 最终输出文件统一命名为 `课程编码-课程名称 课程大纲[26版].docx`

**课程设计/实习实训/毕业论文：** 使用基于模板的 docx 生成策略：
   - 从技能目录找到对应课程性质的模板 .docx 文件
   - 复制到工作区，重命名为 `课程编码-课程名称 课程大纲 模板.docx`
   - 编写 Python 脚本，用 `SyllabusFromTemplate` 类打开模板并填充内容
   - 运行脚本，生成最终 `课程编码-课程名称 课程大纲[26版].docx`

> 所有模板文件本身已含正确的页眉信息表、页边距、字体和标题结构。不要从零创建文档，始终以模板为基础。

### 3.7 docx 生成方法

每次生成大纲时，必须同时提供 .docx 格式。方法因课程性质不同而分为两种路径。

#### 3.7.1 理论课程：基于示例大纲替换内容

理论课程使用 `示例：机械原理 课程教学大纲.docx` 作为基底（该文件已包含完整的页眉表、评分标准系列表 Table 3-11 等结构），避免从零构建复杂表格。具体步骤：

**步骤一：复制示例文件到工作区**

```python
import shutil, os
skill_dir = r"技能目录路径"
work_dir = r"工作目录路径"
src = os.path.join(skill_dir, "示例：机械原理 课程教学大纲.docx")
dst = os.path.join(work_dir, "课程编码-课程名称 课程大纲 模板.docx")
shutil.copy(src, dst)
```

**步骤二：基于 Markdown 大纲内容替换 docx**

打开复制后的 docx 文件，用 python-docx 逐项替换内容。核心原则：**保留模板的所有表格结构（尤其是评分标准系列表的行列结构），仅替换其中的文本内容。** 示例大纲中 12 个表格的分布如下：

| 表索引 | 内容 | 操作方式 |
|:---:|:---|:---|
| Table 0 | 页眉信息表（9行x11列） | 更新各单元格为当前课程的元数据 |
| Table 1 | 课程目标与毕业要求对应关系 | 替换表头和行数据 |
| Table 2 | 教学内容与课程目标对应关系 | 替换行数据 |
| Table 3 | 平时作业与课程目标对应关系 | 替换行数据 |
| Table 4 | 平时作业评分标准 | 替换行数据（保持评价等级结构） |
| Table 5 | 课堂测验与课程目标对应关系 | 替换行数据 |
| Table 6 | 课堂测验评分标准 | 替换行数据 |
| Table 7 | 实验与课程目标对应关系 | 替换行数据 |
| Table 8 | 实验评分标准 | 替换行数据 |
| Table 9 | 期末考试与课程目标对应关系 | 替换行数据 |
| Table 10 | 期末考试评分标准 | 替换行数据 |
| Table 11 | 课程目标达成度评价成绩构成 | 替换行数据 |

核心代码模式（在 Markdown 生成后执行）：

```python
import docx
from docx.shared import Pt

doc = docx.Document("课程编码-课程名称 课程大纲 模板.docx")

# === 全局替换 ===
# 将原文中所有旧课程名称替换为新课程名称
for p in doc.paragraphs:
    for run in p.runs:
        if "机械原理" in run.text:
            run.text = run.text.replace("机械原理", "新课程名称")

# === 替换页眉表（Table 0）===
t0 = doc.tables[0]
# 根据示例大纲的布局填写具体单元格
t0.rows[0].cells[1].text = "课程编码"   # 编码
t0.rows[0].cells[2].text = "课程名称"   # 名称（合并单元格）
t0.rows[0].cells[7].text = "课程负责人" # 负责人
t0.rows[1].cells[1].text = "课程全称"   # R1 课程名称全称
t0.rows[2].cells[1].text = "英文名称"   # R2 英文名称
t0.rows[3].cells[1].text = "学分"       # R3 学分
t0.rows[4].cells[1].text = "执行学期"   # R4 学期
t0.rows[4].cells[3].text = "总学时"
t0.rows[4].cells[5].text = "讲课"
t0.rows[4].cells[6].text = "实验"
t0.rows[4].cells[8].text = "上机"
t0.rows[4].cells[10].text = "实践"
t0.rows[5].cells[1].text = "考核方式"
t0.rows[6].cells[1].text = "授课对象"
t0.rows[7].cells[1].text = "先修课程"
t0.rows[8].cells[1].text = "后续课程"

# === 替换正文段落 ===
# 按顺序替换各节内容（从已生成的 Markdown 中提取文本）
# 找到"一、课程的性质、任务和目标"对应的段落，替换其后内容
for i, p in enumerate(doc.paragraphs):
    text = p.text.strip()
    # 替换各子标题下的正文
    if text == "1. 课程性质":
        # 找到该段落后紧跟的正文段落，替换 text
        pass
    # ... 逐个替换

# === 替换 Table 1（课程目标与毕业要求对应关系）===
t1 = doc.tables[1]
t1.rows[0].cells[0].text = "毕业要求"
t1.rows[0].cells[1].text = "指标点"
t1.rows[0].cells[2].text = "课程目标"
# 替换数据行
for ri, (gr, ip, co) in enumerate(zip(毕业要求列表, 指标点列表, 课程目标列表)):
    row = t1.rows[ri + 1]
    row.cells[0].text = gr
    row.cells[1].text = ip
    row.cells[2].text = co

# === 替换评分标准表（Table 3-11）===
# 每张表的结构一致，只需替换文本内容，保持行列数不变
# 具体替换逻辑根据各表的实际数据动态生成

doc.save("课程编码-课程名称 课程大纲[26版].docx")
```

> **重要原则：**
> - 不要增加或删除模板中的表格行/列，只替换文本。示例大纲的评分标准表格（Table 4/6/8/10）采用"优秀/良好/中等/及格/不及格"五级评价体系，对应新课程时沿用该结构，仅替换评价描述文字。
> - 页眉表（Table 0）中的合并单元格布局复杂，替换时只设置 visible 单元格（如下标 0/1/2/3 等），python-docx 会自动处理合并关系。
> - 正文段落（非表格部分）采用"段落文本匹配 → 替换"的策略：先找到特征段落（如"1. 课程性质"），再替换其后续的正文段落内容。
> - 评分标准中的"课程目标 N"引用必须与新课程的课程目标编号一致（目标1/2/3/4）。

#### 3.7.2 非理论课程（设计/实习/毕设）：基于 SyllabusFromTemplate 生成

课程设计、实习实训、毕业论文三类继续使用 `SyllabusFromTemplate` API 生成。步骤同旧版 3.7 节（基于 `generate_syllabus_docx.py` 脚本）：

**步骤一：复制模板到工作区**

> 注：源文件为旧版 .doc 格式，仅作结构参考。实际输出通过步骤二用 python-docx 创建 .docx 文件。

**步骤二：创建并填充 .docx 版课程简介**

由于源文件为旧版 .doc 格式，需要通过 python-docx 创建一个新的 .docx 文件，其内容结构基于模板中的布局：

课程简介文档结构如下（排版规范同大纲：宋体小四、1.25 倍行距、相同页边距）：

```
课程名称：XXX
课程编码：XXX
课程简介：课程性质、课程定位、教学内容要求，篇幅控制在300-500字
考核形式：考试/考查
建议教材与参考书目：教材名称、作者、出版社
```

填充的数据源为已生成的 Markdown 大纲中"一、课程的性质、任务和目标"及"三、建议教材与学习资源"的内容。

```python
from docx import Document
from docx.shared import Pt, Cm
from docx.enum.text import WD_ALIGN_PARAGRAPH

doc = Document()
# 设置页面 A4、页边距
section = doc.sections[0]
section.page_width = Cm(21.0)
section.page_height = Cm(29.7)
section.top_margin = Cm(2.5)
section.bottom_margin = Cm(2.5)
section.left_margin = Cm(3.2)
section.right_margin = Cm(3.2)

# 标题
p = doc.add_paragraph()
p.alignment = WD_ALIGN_PARAGRAPH.CENTER
run = p.add_run("课程名称  课程简介")
run.bold = True
run.font.size = Pt(14)
run.font.name = '宋体'

# 正文内容
items = [
    ("课程名称：", "XXX"),
    ("课程编码：", "XXX"),
    ("课程简介：", "课程性质、定位、主要内容，300-500字"),
    ("考核形式：", "考试/考查"),
    ("建议教材与参考书目：", "教材名称、作者、出版社"),
]
for label, value in items:
    p = doc.add_paragraph()
    run_label = p.add_run(label)
    run_label.bold = True
    run_label.font.size = Pt(12)
    run_label.font.name = '宋体'
    run_value = p.add_run(value)
    run_value.font.size = Pt(12)
    run_value.font.name = '宋体'

doc.save("课程编码-课程名称 课程简介[26版].docx")
```

## 5. Skill 资源文件

### 核心脚本
- `scripts/generate_syllabus_docx.py` — 模板化 docx 生成器，提供 `SyllabusFromTemplate` 类。在生成 docx 时，将该文件复制到工作区。
  - `SyllabusFromTemplate(template_path)` — 打开模板，填充页眉表，插入内容，保存。
  - 旧版 `SyllabusDocx`（从零创建）仍保留备查，但不再推荐使用。
  - 脚本内置了 `fill_header_theory()` 和 `fill_header_other()` 两个方法，分别对应 9x11 和 3x4 两种页眉表。

### 模板文件
- `示例：机械原理 课程教学大纲.docx` — **理论课程的首选模板**（2026版）。包含完整的 12 个表格：页眉表 + 课程目标矩阵 + 教学内容对应 + 3 项过程考核（作业/测验/实验）的对应关系及评分标准 + 期末考试对应关系及评分标准 + 达成度评价构成表。**理论课程的 docx 输出以此文件为基础，通过替换内容生成。**
- `1. 理论课程教学大纲的格式.docx` — 理论课程备选模板（简版）。页眉表 9行x11列（含学分/总学时/讲课/实验/上机/实践明细）。仅在示例大纲不适合的情况下使用。
- `2. 课程设计教学大纲的格式.docx` — 课程设计类模板。页眉表 3行x4列（简洁型）。
- `3. 实习教学大纲的格式.docx` — 实习实训类模板。页眉表 3行x4列。
- `5. 毕业论文（设计）教学大纲的格式.docx` — 毕业论文/设计类模板。页眉表 3行x4列。

> 理论课程优先使用 `示例：机械原理 课程教学大纲.docx`（含完整评分标准表），仅在该模板结构不适用于当前课程时才回退使用 `1. 理论课程教学大纲的格式.docx`。
