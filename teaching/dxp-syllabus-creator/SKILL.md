# 高校课程大纲智能编制专家

## 1. 概述

为高校教务人员及教师提供符合 2026 版培养方案的课程教学大纲编制服务。以**专业名称**与**课程名称**为输入，自动识别课程性质，匹配四类模板（理论/设计/实习/毕设），生成 Markdown + .docx 双格式大纲，理论课程额外生成课程简介。

**输出文件命名规范**：
- 大纲文件：`课程编号 课程名称 课程教学大纲（26版）.docx`（注意：括号为全角（26版））
- 课程简介文件：`课程编号 课程名称 课程简介(26版).docx`（注意：括号为半角(26版)）

**排版规范**：各部分对齐方式、字体、字号等格式须与参考模板保持完全一致。理论课程参考模板使用命名样式体系（节标题=黑体12pt加粗、大纲正文=宋体10.5pt两端对齐、表格标题=加粗居中），替换内容时须保留模板原有样式，不可擅自更改。详见 `references/docx_generation_guide.md`。

## 2. 工作区准备与用户确认

### 2.1 检查培养方案
扫描工作目录，确认存在培养方案 .docx 文件。若不存在，提示用户放置。若存在，用 python-docx 解析提取课程元数据（编码、名称、学分、学时、考核方式、学期等）。详见 `references/培养方案解析指南.md`。

### 2.2 教材信息确认
解析培养方案确认课程存在后，**必须主动确认教材**。若用户未提及，给出该课程常见教材选项供选择。教材信息影响「建议教材」节和教学内容章节编排。

### 2.3 考核比重确认（理论课程）
理论课程必须确认期末与过程考核比重。默认：期末 50% + 过程 50%（过程占比须在 30%-50%）。非理论课程按各自模板考核体系生成，无需询问。

### 2.4 工程认证矩阵检查
检查工作目录是否有独立的工程认证矩阵 xlsx 文件。若有，优先从中提取细分指标点支撑关系（如 1.2、2.3 等）；若无，回退到培养方案中的粗粒度矩阵。毕业要求指标点描述从培养方案段落中逐条提取原文，直接填入 Table 1，禁止使用模板中的旧描述。

**解析失败时的用户输入回退**：如果上述途径均未能成功解析到目标课程的工程认证细分支撑点信息（例如培养方案中未收录该课程的矩阵、xlsx 文件缺失或格式不符），**必须主动提示用户输入**该课程对应的工程认证支撑点。可能存在多条支撑点，每条须包含指标点编号和描述原文。提示时给出格式示例：

> 请输入本课程对应的工程认证毕业要求指标点（可能有多条），每条格式为"编号 描述"，多条之间换行分隔。示例：
> 2.2 能利用物理、信息、控制、计算机等相关科学原理正确表达复杂工程问题。
> 3.4 解决方案设计中，能够考虑社会、健康、安全、法律、文化以及环境等影响。

用户输入后，将各条指标点编号和描述原文直接填入 Table 1，与课程目标逐一对应。不可省略此步，也不可用模板中的旧指标点描述替代。

## 3. 课程性质识别与模板绑定

| 性质 | 判定依据 | 模板文件 | 内容侧重 |
|:---|:---|:---|:---|
| 理论课程 | 知识传授为主，实验服务于理论验证 | `XD26010006 供配电与照明工程 课程教学大纲（2026版-参考模版）.docx` | 知识体系、思维方法、AI辅助资源 |
| 课程设计 | 综合应用、方案设计、有任务书 | `2. 课程设计教学大纲的格式.docx` | 方案优化、工程实现、数字孪生 |
| 实习实训 | 现场实践、岗位轮训、技能操作 | `3. 实习教学大纲的格式.docx` | 岗位认知、技能训练、VR/AR |
| 毕业论文 | 含开题、中期、答辩全流程 | `5. 毕业论文（设计）教学大纲的格式.docx` | 科研规范、创新探索、文献检索 |

混合型课程（既有理论又有实验学时）：若实验服务于理论验证而非独立设计任务，判为理论课程。

## 4. 章节安排方案推荐

生成大纲前，**主动分析教材章节与学时**，向用户推荐 2-3 种章节安排方案：

1. **分析教材**：确认教材章节结构，列出各章核心知识点
2. **核对学时**：讲课学时 + 实验学时 = 总学时，确保方案中学时分配合理
3. **参考其他高校**：搜索同类课程的教学安排，了解常见学时分配
4. **推荐方案**：提出 2-3 种方案（如：按教材完整展开/合并为教学单元/精简模块），列明每章学时
5. **用户确认**：等用户选择或自定义后，再进入内容生成

推荐方案时考虑：Table 2 只有 5 行数据空间，若章节数 >5 需合并。学时分配需与培养方案的总学时/讲课/实验完全一致。

## 5. 八大编制维度

1. **元数据锁定**：编码、名称、总学时严格对齐培养方案
2. **先修课程**：基于培养方案课程计划表的前后关系确定
3. **性质与任务**：按课程性质差异化表述，支撑毕业要求指标点
4. **内容与重难点**：学时与培养方案一致；重难点具体到概念/算法名称
5. **能力培养**：聚焦"创新思维+实践能力+数字素养"
6. **教材与资源**：优先规划教材；线上资源须含 ≥2 条国外头部高校公开课链接（MIT/Stanford/Berkeley 等），链接真实可访问；数智资源明确到工具/平台名称
7. **考核与成绩**：过程评价 30%-50%，权重总和 100%；标注支撑的课程目标编号
8. **教改摘要**：四维（立德树人/教学思想/教学方法/教学手段）均须结合课程具体内容生成，不可空泛

## 6. Markdown 大纲结构

标题层级：`#` 文档标题、`##` 六节主标题、`###` 子标题。

### 元数据表（第一节之前）

```
# XXX 课程教学大纲

| 课程编码 | XXX | 课程名称 | XXX |
| 英文名称 | XXX | 适用专业 | XXX |
| 课程性质 | XXX | 学分 | XXX |
| 总学时 | XXX | 讲课 | XXX |
| 实验 | XXX | 上机 | XXX |
| 实践 | XXX | 考核方式 | XXX |
| 适用学期 | XXX | 先修课程 | XXX |
```

总学时/讲课/实验/上机/实践必须各自占独立行，纯数字，不合并。

### 六节标题

- `## 一、课程的性质、任务和目标`（理论课；设计/实习/毕设替换"课程"为对应类型）
- `## 二、课程教学内容的基本要求、重点和难点及学时分配`
- `## 三、建议教材与学习资源`
- `## 四、考核形式与成绩构成`
- `## 五、有关说明`
- `## 六、课程建设与改革摘要`

### 第一节：课程目标格式

每条课程目标必须使用 `**课程目标N：**` 格式开头，描述紧跟冒号，句尾标注 `【支撑毕业要求X.Y-强度】`。从培养方案支撑的指标点逐条推导，一般 3-4 个，涵盖知识/能力/素养。

### 第二节：章节三要素

每章必须包含三要素，不可简化：

- **基本要求**：用"了解/理解/掌握"三级动词，至少 3 条具体要求
- **重点和难点**：具体到概念/协议/算法名称
- **教学方法**：说明具体方法组合及本章应用方式；有实验学时的章节标注实验名称和工具

末尾附 Table 2（教学内容与课程目标对应关系），"课程教学内容"列每行 3-5 个核心知识点，用"；"分隔。

### 第三节：教材与资源排序

- **建议教材**：仅放用户确认的一本主教材
- **参考教材**：≥3 本，按学术水平排序
- **线上学习资源**：≥2 条国外头部高校公开课 + 国内 MOOC，链接真实可访问
- **数智化教学资源**：具体工具/平台名称 + 用途 + 链接（不使用学习平台名称作为资源条目）

### 第四节：考核

含 Table 3（考核项目对照表）。过程考核（作业/测验/实验）和期末考试各自列出与课程目标的对应关系及评分标准表。

### 编制说明与合规自查报告（附录）

用 `---` 分隔后附：专业与课程匹配说明、课程性质判定依据、工程认证支撑关系、现有大纲继承对照表（如有）、参考资源清单、八大维度合规自查结果。

## 7. 第五节和第六节编写指南

### 第五节：有关说明（必填，不可空）

至少包含：先修课程衔接、后续课程承接、开课学期建议及理由、修读建议、其他说明。

### 第六节：课程建设与改革摘要（四维均须完整，每维 ≥150 字）

1. **立德树人**：挖掘课程专属思政元素（具体知识点→思政内容→融入路径→育人成效），避免通用套话。模板批注要求"课程思政要紧密结合课程内容，建议每门课至少选 3 个典型思政提要融入课程内容和教学方法"——生成时须确保至少 3 个思政融入点，且与课程具体章节内容绑定
2. **教学思想**：须体现《教育发展"十五五"规划》精神，包含以下要点（可融入课程特色表述，但核心思想不可省略）：
   - 随着人工智能特别是生成式 AI 的快速普及，知识获取的门槛被大幅降低，传统课堂以知识讲授为核心的优势正在不断减弱，单向灌输式教学的局限日益显现
   - 高校课堂亟须从"教知识"向"育能力"转型，推动人才培养由单一知识传授向综合能力提升转变
   - 强化学生核心知识、核心能力培养，推广"项目制"育人培养方式
   - 着力培育学生的批判性思维及解决复杂问题能力
   - 结合 OBE 落实做法、学生中心体现、知识/能力/价值三者统一
3. **教学方法**：2-4 种具体方法及适用场景，"理论+实践+创新"三维落地。模板批注说明"涉及教学方法均为建议，可根据教学内容、教学手段实时更新、调整"——生成的方法应贴合课程实际，非生搬硬套
4. **教学手段**：具体智慧教学平台名称 + 具体AI/仿真工具名称 + "课前-课中-课后"闭环 + 数据分析

四维侧重不同：立德树人=育人价值、教学思想=理念导向、教学方法=怎么教、教学手段=用什么工具。

## 8. docx 生成方法

### 8.1 理论课程：基于参考模板替换内容

从技能目录复制 `XD26010006 供配电与照明工程 课程教学大纲（2026版-参考模版）.docx` 到工作区，用 python-docx 替换内容。模板含 10 个表格（页眉表 + 课程目标矩阵 + 教学内容对应 + 平时作业对应 + 平时作业评分 + 随堂测试对应 + 实验对应 + 实验评分 + 期末考试对应 + 达成度评价）。评分标准表采用 4 级制（优/良/中及格/差），不是 5 级制。

**核心原则**：保留模板所有表格结构和命名样式，仅替换文本内容。不可更改段落的样式名（如"节标题""大纲正文""大纲强调"等），不可更改字体、字号、对齐方式。

**关键步骤**：
1. 复制模板到工作区
2. 安全全局替换（只替换完整课程名，不替换单字）
3. 填充页眉表（保护标签单元格和负责人区域）
4. 替换第一节正文（课程性质/任务/目标）
5. 替换第二节正文（各章节三要素）——**注意：模板只有一组章节占位段落，第 2 章及之后需用 `insert_paragraph_before()` 插入新段落**
6. 替换建议教材段落（仅保留一本主教材）
7. 替换第五节和第六节正文
8. 填充 10 个表格数据
9. 调整列宽
10. 保存为 `课程编号 课程名称 课程教学大纲（26版）.docx`

**详细规则**（页眉表单元格映射、评分标准表格式、段落替换策略、多章节插入方法、常见陷阱）见 `references/docx_generation_guide.md`。

**工具函数**：`scripts/generate_syllabus_docx.py` 提供以下函数（理论课程替换用）：
- `add_run_songti(p, text, bold)` — 添加 run 并设置宋体字体
- `set_cell_font(cell, text, align)` — 安全设置单元格文本
- `insert_paragraph_before(ref_p, text, bold)` — 在段落前插入新段落
- `safe_replace_course_name(doc, old, new)` — 安全全局替换课程名
- `fill_header_table(table, data_map)` — 填充页眉表（自动保护标签）
- `fill_table_from_data(table, data, align_map)` — 用数据数组填充表格
- `replace_section_body(doc, keyword, body)` — 找标题替换正文
- `clear_excess_rows(table, start_row)` — 清空多余行
- `adjust_grading_table_widths(table)` — 调整评分表列宽

### 8.2 非理论课程：SyllabusFromTemplate

课程设计/实习/毕设使用 `SyllabusFromTemplate` 类（同一脚本文件中），复制对应模板，填充页眉表（`fill_header_other()`），设置节标题，从后向前插入内容。

### 8.3 课程简介（理论课程必做）

用 python-docx 创建新 .docx，结构：标题 + 课程名称/编码/简介(300-500字)/考核形式/建议教材与参考书目。排版同大纲。保存为 `课程编号 课程名称 课程简介(26版).docx`（注意：括号为半角）。

### 8.4 模板批注编制要求（理论课程）

参考模板 `XD26010006 供配电与照明工程 课程教学大纲（2026版-参考模版）.docx` 中含 5 条批注，是模板编制方对大纲内容的具体要求。**生成 docx 时不生成批注本身**，但大纲内容必须遵循这些要求：

| 批注位置 | 批注内容 | 生成时须遵循的要求 |
|:---|:---|:---|
| 教学内容区域 | 教学内容应该具体明确，重点难点突出！课程思政要紧密结合课程内容，建议每门课至少选 3 个典型思政提要融入课程内容和教学方法 | 第二节各章"基本要求"用"了解/理解/掌握"三级动词且 ≥3 条；"重点和难点"具体到概念/算法名称；立德树人部分至少 3 个思政融入点与课程章节绑定 |
| 教学方法 | 涉及教学方法均为建议，可根据教学内容、教学手段实时更新、调整 | 各章"教学方法"应贴合课程实际内容，非生搬硬套；可结合课程特点灵活调整 |
| 表 2（教学内容与课程目标对应） | 教学内容和课程目标应根据实际情况对应，建议可以对应到章、节，建议细化到知识点 | Table 2 "课程教学内容"列每行 3-5 个核心知识点，用"；"分隔；对应关系细化到章节/知识点级别 |
| 表 4（平时作业评分标准） | 此评分标准适用于按照等级划分的作业成绩；如果作业成绩已经量化，按照实际分数折算。同样适用于随堂测验 | 评分标准表按 4 级制（优 90-100/良 80-89/中及格 60-79/差 0-59）生成；如课程采用量化评分，在评分标准中说明折算方式 |
| 期末考试 | 期末考试如果采用闭卷、开卷考试，评分标准按照试卷评分标准；如果采用大作业、论文等无法具体量化的形式，需要给出相应的期末成绩评价等级标准 | 期末考试评分标准根据考核形式（闭卷/开卷/大作业/论文）给出对应的评分方式；非闭卷形式须给出等级评价标准 |

### 8.5 格式一致性要求（理论课程）

生成的 docx 各部分格式须与参考模板保持一致，具体如下：

| 内容部分 | 样式名 | 字体 | 字号 | 对齐 | 其他 |
|:---|:---|:---|:---|:---|:---|
| 文档标题 | Heading 1 | Times New Roman + 黑体 | 14pt（运行级覆盖） | 居中 | 加粗 |
| 节标题（一~六） | 节标题 | Times New Roman + 黑体 | 12pt | 两端对齐 | 加粗，1.5 倍行距 |
| 子标题（课程性质/基本要求等） | Normal | Times New Roman + 宋体 | 12pt | 两端对齐 | 加粗 |
| 正文段落 | Normal / 大纲正文 | Times New Roman + 宋体 | 12pt（运行级）/ 10.5pt（样式） | 两端对齐 | 1.5 倍行距 |
| 表格标题（表1~表9） | Body Text Indent / 表格标题 | 宋体 | — | 居中 | 加粗 |

**关键原则**：替换文本内容时保留段落原有样式名和格式属性，仅通过 `p.clear()` 清除 run 文本后用 `add_run_songti()` 写入新文本。不要更改段落的 `style` 属性、`alignment` 属性或 `paragraph_format` 属性。新插入的段落应复制参考段落的样式。

## 9. 编码注意事项

### 中文字体一致性
参考模板使用双字体体系：西文字体为 Times New Roman，中文字体（eastAsia）为宋体；节标题和文档标题的中文字体为黑体。`p.clear()` 后 `add_run()` 创建的 run 不继承模板字体。所有正文替换必须用 `add_run_songti()` 函数，显式设置 `font.name='Times New Roman'`（西文）和 `eastAsia='宋体'`（中文）。节标题替换须设置 `eastAsia='黑体'`。

### 中文引号
Python 字符串中的中文引号（""）可能与 ASCII 双引号混淆导致语法错误。解决方案：用 `\u201c`/`\u201d` 转义，或用三引号字符串。

### PowerShell 编码
PowerShell 默认 GBK 编码，运行 Python 时中文可能乱码。在脚本头部加 `sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')`。避免在 PowerShell 中内联 Python（f-string 引号冲突），改为写入 .py 文件后执行。

### 课程目标编号一致性
全文（段落、表格、评分标准）统一使用"课程目标N"格式（含"课程"前缀），禁止混用"目标N"等变体。
